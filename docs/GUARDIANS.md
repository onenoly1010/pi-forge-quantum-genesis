## Becoming a Guardian

### Requirements
1. **Active Contributor**: Minimum 10 merged pull requests
2. **Covenant Adherent**: Demonstrated understanding of principles
3. **Community Standing**: Positive interaction history

### Nomination Process
1. Self-nomination or peer nomination via issue
2. 14-day community feedback period
3. Existing guardian vote (unanimous required)
4. Onboarding and mentorship period

## Guardian Rotation Schedule

Guardians rotate primary on-call responsibility weekly:
- **Week N**: Primary handles triage and urgent reviews
- **Week N+1**: Backup role, supporting primary
- **Week N+2**: Focus on long-term improvements

See [Ritual Calendar](./RITUAL_CALENDAR.md) for current rotation.

## Escalation Protocols

### Severity Levels
1. **Low**: Style/documentation issues → Direct PR comment
2. **Medium**: Covenant minor violations → Guardian discussion
3. **High**: Security/ethics concerns → Immediate escalation to all guardians
4. **Critical**: Active exploitation → Emergency response protocol

### Emergency Contact
For critical security issues, contact @onenoly1010 directly via GitHub security advisory.
